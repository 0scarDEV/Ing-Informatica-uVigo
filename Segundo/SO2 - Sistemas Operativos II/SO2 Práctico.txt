SO2 Práctico:

-Práctica 1 : Teoría

1. Elección de un sistema operativo (antes de instalar).

1.1 Requisitos de hardware

CPU: arquitectura (32/64 bits), compatibilidades e instrucciones. Más potencia = mejor respuesta.

Núcleos: más núcleos = mejor multitarea.

RAM: si falta RAM, se usa disco como memoria virtual y el rendimiento cae mucho.

Disco: espacio y velocidad (SSD mejora arranque y carga).

Compatibilidad: drivers y soporte para gráfica, red, RAID, periféricos.

1.2 Características del equipo

Identificar discos y particiones existentes.

Ver si hay SO previos y si habrá arranque dual.

Planificar particiones necesarias (en Linux es típico separar /, /home y swap).

1.3 Gestor de arranque

Permite elegir qué SO iniciar cuando hay varios.

En Linux, lo habitual es GRUB, que detecta otros sistemas (ej. Windows).

1.4 Licencias

Ver si el SO y el software requieren licencias (libre, comercial, académica).

Coste y número de equipos puede ser decisivo.

1.5 Copias de seguridad

Antes de tocar particiones o instalar, hacer backup para evitar pérdidas.

1.6 Software a instalar

El software condiciona el SO y los recursos necesarios (versiones, requisitos).

1.7 Separar sistema y datos en particiones
Ventajas:

Facilita backups.

Permite reinstalar el SO sin perder datos.

Mejor organización/mantenimiento.
Inconveniente:

Hay que decidir bien el tamaño de cada partición.

1.8 Métodos de instalación soportados

Desatendida: automatizada con ficheros de configuración.

Por red: instalación desde servidor vía red.

Clonado: copiar imagen a otros equipos (hardware similar).

Live: probar sin instalar en disco.

2. Virtualización del sistema operativo

2.1 Qué es

Permite ejecutar varios sistemas operativos a la vez en un mismo equipo.

Se usan máquinas virtuales (VM) que emulan un ordenador completo.

Las gestiona un software de virtualización llamado hipervisor.

2.2 Host y Guest

Host (anfitrión): el sistema operativo del equipo físico.

Guest (invitado): el sistema operativo dentro de la VM.

Un host puede tener varios guests.

2.3 Requisitos hardware para virtualizar

CPU con extensiones de virtualización: Intel VT-x / AMD-V.

RAM suficiente para repartir entre host e invitados.

Disco suficiente.

Soporte de virtualización en CPU/placa mejora rendimiento.

3. Tipos de virtualización (por cómo se integra el hipervisor)

3.1 Bare-Metal (Tipo 1)

El hipervisor se instala directamente sobre el hardware (sin SO anfitrión).

Rendimiento cercano al nativo.

Más orientado a servidores/producción y suele aportar más seguridad.
Ejemplos: VMware ESXi, Xen, Citrix XenServer, Hyper-V Server, Oracle VM Server.

3.2 Hosted (Tipo 2)

El hipervisor es una aplicación sobre un SO anfitrión.

Más fácil para casa/educación.

Menor rendimiento que tipo 1 porque el host gestiona los recursos.
Ejemplos: VirtualBox, VMware Workstation, Virtual PC.

4. Contenerización (relacionada pero no igual que VM)

Aísla aplicaciones y dependencias en entornos ligeros (contenedores).

No necesita un SO invitado completo → menos consumo de CPU/RAM y despliegue rápido.

Menor aislamiento que una VM, pero muy eficiente para muchas apps.
Ejemplo típico: Docker.



-Práctica 2 : Teoría

1. Linux es multiusuario: por qué importa:

Linux permite que trabajen varios usuarios en el mismo sistema. Para que sea seguro y estable, usa cuentas, permisos y grupos para controlar qué puede hacer cada uno y proteger tanto los datos como el sistema.

2. Tipos de cuentas:

2.1 root (superusuario, UID 0)
Control total del sistema. En Debian suele estar activa con contraseña; en Ubuntu suele venir deshabilitada y se administra mediante sudo.

2.2 Usuarios normales
Uso diario. Permisos limitados, normalmente restringidos a su directorio personal y directorios compartidos como /tmp.

2.3 Cuentas de sistema/servicios
No representan personas, sino servicios (web, correo, demonios…). Se usan para aislar privilegios. No deberían permitir inicio de sesión: contraseña bloqueada y shell de “no login”.

3. Grupos: para qué sirven:

Permiten gestionar permisos de forma colectiva.
Cada usuario tiene un grupo primario y puede pertenecer a varios grupos secundarios.
Un grupo puede contener usuarios, pero no otros grupos.

4. Archivos clave de usuarios y grupos

4.1 passwd
Guarda datos básicos del usuario: nombre, UID, GID, comentario, home, shell. La contraseña no está aquí (aparece una x).

4.2 shadow
Guarda contraseñas cifradas y políticas. Solo accesible por root. Si la contraseña está bloqueada suele aparecer ! o *.

4.3 group
Guarda la información de grupos: nombre, GID y lista de usuarios secundarios (separados por comas). El grupo primario no se refleja aquí porque se define en passwd.

4.4 gshadow
Similar a shadow pero para grupos (políticas, administradores, etc.).

4.5 skel
Directorio plantilla que se copia al home de los nuevos usuarios (archivos de configuración inicial).

5. Gestión de grupos (comandos):

-Crear grupo: groupadd (o addgroup en Debian).

-Eliminar grupo: groupdel (o delgroup en Debian). No se puede borrar si algún usuario lo tiene como grupo primario.

-Modificar grupo: groupmod (cambiar GID o nombre).

-Grupos típicos para permisos: cdrom, audio, dialout, video, lp, plugdev.

6. Gestión de usuarios (comandos)

6.1 adduser (Debian, interactivo, “hace todo”)

Crea el usuario y su grupo.

Asigna grupo primario.

Crea home.

Copia plantilla inicial.

Asigna shell por defecto (bash).

6.2 useradd (más técnico, configurable)

Permite especificar home, crear directorio, shell, etc.

No asigna contraseña automáticamente → hay que usar passwd.

6.3 borrar y modificar

Borrar usuario: userdel (con -r también borra su home).

Modificar usuario: usermod (cambiar grupo primario, home, shell, comentario, login…).

6.4 pertenencia a grupos

Añadir a grupo secundario: usermod -a -G (o adduser usuario grupo en Debian).

Quitar de grupo: se redefine la lista con usermod -G o se usa deluser usuario grupo.

7. Cambio de usuario: su vs su -

su usuario: cambia usuario sin cargar el entorno completo.

su - usuario: conmutación completa (como un login), carga home/variables/shell.

Volver al usuario anterior: exit.

8. Privilegios con sudo

Permite ejecutar comandos como root usando tu contraseña (si estás autorizado).

Para habilitarlo en Debian: añadir el usuario al grupo sudo.

Hay un período de gracia (no pide contraseña cada vez); se puede invalidar con sudo -k.

Ver permisos: sudo -l.

Ver permisos de otro usuario: sudo -l -U usuario.

Conmutar a root con sudo:

sudo su (mantiene entorno del usuario)

sudo su - (entorno completo de root)

sudo -i (recomendado: shell interactiva de root “limpia”)

9. sudoers y visudo (clave)

La configuración de sudo se define en el archivo sudoers.

Debe editarse con visudo porque bloquea edición simultánea y valida sintaxis para no romper sudo.

-Práctica 3 : Teoría

Puntos de estudio de esta práctica:

1."Todo es un archivo" y porque /dev no es una carpeta normal.
En linux el kernel tiene muchos archivos especiales en /dev.
En /dev no hay datos como en un fichero normal, sino interfaces (nodos de dispositivos) con un major code que indica el tipo de dispositivo(audio, video ...)
y el minor que da detalles más concretos sobre el(altavoz, auricular ...).

2.Dispositivos de bloque vs dispositivos de carácter.

2.1 Los dispositivos de bloque:
Se accede por bloques de tamaño fijo.
Permite acceso aleatorio.
Ejemplos: USB, SSD...

2.2 Los dispositivos de carácter:
Flujo secuencial(carácter a carácter).

3.Comandos.
lsblk: Te muestra el árbol del disco, particiones y cosas cómo size, type, mountpoint...

sudo fdisk -l: Proporciona información detallada sobre las particiones, tabla de
particiones (MBR o GPT) y tamaños. Es útil para un diagnóstico
técnico preciso.

df -h: Indica qué particiones están montadas actualmente y su nivel de
ocupación, en un formato orientado al uso del almacenamiento. La
opción -h muestra los tamaños en unidades legibles por humanos
(MiB, GiB).

du:  tamaño de directorios y ficheros

fsck: comprobación y reparación de sistemas de archivos

4.Particionado: qué es y para que sirve.
Particionar es dividir el disco en "trozos" de manera lógica para poder:
Crear sistemas de archivos separados.
Crear swap(Se verá más adelante que es y para que sirve).
Reservar particiones de arranque.


5.Estándares de tablas de particiones(MBR VS GPT).

5.1 MBR
Máximo 4 particiones.
Para más usas 1 extendida cómo "contenedor" y dentro lógicas.
Límite del tamaño del disco: 2TB.

5.2 GPT
Soporta discos enormes.
Hasta 128 particiones típicamente(aunque depende de implementación).
Guarda copias de la tabla de particiones.
Es el estándar con UEFI.
Recomendado hoy en día casi siemppre.


6.Tipos de particiones.
Primaria: Partición directa, en MBR cómo máximo 4.
Extendida: Sólo en MRB, es un contenedor con particiones lógicas.
Lógica: Partición dentro de una extendida.
*Especiales*
ESP.
SWAP.
BIOS BOOT.

7.Herramientas de particionado.
fdisk: muy usado tradicionalmente para MBR(también se usa en GPT).
gdisk: orientado a GPT.
parted: potente.
cfdisk: interfaz más amigable.

8.Sistemas de archivos, qué es y ejemplos.
Un sistema de archivos define cómo se organizan directorios y archivos, cómo se guardan metadatos
y mecanismos de fiabilidad.
Comunes: ext4(estándar en Linux), XFS...

9.Swap.
Swap = espacio en disco duro usado cómo apoyo de RAM.
Dos formas:
1.Partición swap
2.Archivo swap

10.Dispositivo loop: montar un archivo cómo si fuera un disco.
Un loop device permite tratar un archivo cómo dispositivo de bloque.

1.Expresiones regulares:

Una expresión regular es una secuencia de caracteres que conforman un patrón de búsqueda.
Ejemplo: Buscar en un archivo dde texto las lineas con la palabra amarillo, que puede estar escrito
con a mayúscula o minúscula y que puede que esté en plural.

En este tema veremos que se puede hacer con este comando:

Metacarácter: Caracteres con significado especial como . que significa cualquier carácter.

Estos funcionan exactamente igual con grep y con grep -E. Siempre son especiales y nunca llevan barra (a menos que quieras buscarlos literalmente).

Metacaracteres "Cambiantes".

( ) : Grupos/subexpresiones : Con grep(básico) necesita barras \( \) : Con grep -E sin barra.

| : OR(Uno o otro) : Con grep(básico) necesitas barra (grep 'A\|B') : Con grep -E sin barra (grep -E 'A|B').

-Práctica 5 : Teoría

SED es la abreviatura de stream editor. Opera sobre un flujo de texto que recibe como entrada
y que envía transformado a la salida.
Cómo comando de Linux va a operar sobre un archivo de entrada, recibiendo y operando linea a linea
para despues enviarla o enviarlas cara otro archivo o a la salida estándar.

Es un comando muy útil para el admin del sistema porque le permite hacer programas para procesar
archivos de texto.

En el sed cada línea va entrando en el espacio de patrones donde se le aplican los cambios.
Sed  t ambién tiene otro epacio llamado espacio de almacenamiento que actúa cómo zona de memoria
de manera que el espacio de patrones puede ser almacenado ahí y recuperado posteriormente.
Finalmente el contenido del espacio de patrones será enviado a la salida antes de procesar
la siguiente línea.

Sintaxis básica : sed [opciones] [-e 'script'] [-f programa] file(s).

Ejemplo simple:sed -e '11,$d' file.txt
-e indica que lo que sigue es el script a ejecutar.
11,$ indica que de a fila 11 a la última.
d inddica que sobre el anterior rango se borre lo que abarque.

Opciones interesantes del sed:
-n: Anula la salida automática y las salidas se harán con el comando p que se verá más adelante.
-r|E : expresiones regulares extendidas.
-i: sobreescribe el archivo SIN crear copia de seguridad.
-i.bak: crea copia de seguridad.
-s: si pasamos varios archivos se tratan de forma independiente y no cómo un todo.
-e: Para indicar que lo que viene son las instrucciones, pero si las separas en caso de poner
más de una con ; es opcional.
-E: expresiones regulares extendidas, las de siempre menos {}, es decir, + ? | ().

En cuanto al script, la sintaxis básica sería [rango], intrucciones, [argumentos]
Ejemplo:sed -e '1,5s/error/CORREGIDO/g' log.txt de la linea 1 a la 5 si ves error cambialo por corregido en toda la linea(g).

Lo primero de todo que se ve es el rango, sino lo dan se aplica todas las líneas. Si se necesitan
usar varias expresiones se separan con ;. Si sólo ponemos un número enel rango estamos diciendo
que queremos que sólo se aplique a esa linea. Un caso particular sería $ que se refiere a la 
última línea. Si queremos indicar un rango de líneas separamos con ,.

También puede aparecer aquí expresiones regulares, en ese caso tienen que ir entre / /. Ya que no
es un comando, es un patrón a buscar.

Ejemplo: Todas las líneas que emiecen por a se borran.
sed -e ’/^a/d’ file.txt

Por último, ! indica lo contrario.
Por ejemplo aquí, todas las lineas menos la 20 se borran.
sed -e '20!d' file.txt.

Instrucciones:

1.d(Borrado).
Ejemplo: sed -e ’/^a/,$d’ file.txt.
Borrar desde la primera línea que empiece por a, hasta el final.

2.s/patron/novo/flag.
Uno de los comandos más usados, la s indica substitución.
Busca un patrón y si lo encuentra lo substituye por lo que le dices.
Ejemplo: sed -r ’s/[0-9]+/N/’ file.txt.
Si encuentra uno o más números seguidos los substituye por una N.
Así sólo lo haría para el primer número que aparezca en la línea. Si lo queremos para  toda
debemos usar el flag g.
sed -r ’s/[0-9]+/N/g’ file.txt.

Otro de los flags útiles en esta instrucción es p que hace print.
Un ejemplo para entenderlo:
sed -n -e ’s/^[0-9].*[0-9]$/liña con números/p’ file.txt
Invocamos sed con -n que como recordamos suprime la salida automática. Después vemos la 
instrucción de substitución que substituye las líneas que empiezan por un número, tienen
cosas random en medio y acaban en número por "líneas con números". Y ademas imprimirá un línea de
números por cada línea que cumpla el patrón.

Otro ejemplo interesante y más completo: 
sed -n -r ’s/^[0-9]+([^0-9].*[^0-9])[0-9]+$/\1/p’ file.txt.
Tenemos -n para desactivar la salida automática y -r para expresiones regulares extendidas.
Luego buscamos líneas que cumplan el siguiente patrón:
Que la línea empiece por un o más números, seguidos de un NO número seguido de uno o más 
caracteres cualesquiera seguidos otr vez de un NO número y rematado en uno o más números.
Todo eso se sustituye por el "seguidos de un NO número seguido de uno o más 
caracteres cualesquiera seguidos otr vez de un NO número" y se imprimen las líneas donde
se cumpla el patrón y por tanto se substituya una cadena por la otra. Finalmente se enseña
lo cambiado por p.
Si en vez de \1 pusieramos &, enseñaría toda la línea.(Ignoraría s).


3.q Indica que hai que rematar o procesado das liñas e saír.
Ejemplo: sed -e ’/^$/q’ file.txt
Si encuenta una línea vacía acaba el procesado y sale.
Por lo tanto enseña todas las líneas hasta la primera vacía.


4. Enviar a liña cara a saída. Enviar a liña cara a saída.(Acompañado de -n al principio).


5.n Copia o contido do espazo de patróns na saída e carga a seguinte liña.
Ejemplo: sed -e ’/^[0-9]/{n;s/[0-9]/x/g}’ file.txt

Importancia de {} aquí, para agrupar instrucciones.
Este comando busca líneas comenzadas por número y la envía a salida y carga la siguiente.
Y e n esa cambia sus números por x.
Si quitas las llaves la segunda instrucción se le hace a todas las líneas.



5.y substitúe un conxunto de caracteres por outro. Vexamos un exemplo:
sed -e ’y/aeiou/AEIOU/
Si encuentra una vocal minúscula la cambia a mayúscula.


6.a,i,c Engade (append), inserta (insert) ou cambia (change) un texto despois,
antes ou no canto da liña actual. Van un par de exemplos:

sed -r ’/[0-9]+/i\---’ file.txt 
añade --- antes en las lineas con números.

sed -e ’10 c\liña 10 suprimida’ file.txt
cambia la linea 10 por linea suprimida(sobreescribe).


Control de flujo de entrada:

n: Básicamente, uno sí y otro no.
Lee uno, le da salida, carga el otro y ahí aplica el patrón.
Ejemplo: sed '/PATRON/{n; s/A/B/}' archivo
	cuando encuentra una línea con patrón a esa no le hace nada y cambia la siguiente.

N: Anexa la nueva línea al espacio de patrones actuales con la que ya hay(va de 2 en 2).
sed '/\\$/{N; s/\\\n//}' datos.txt
Las líneas acabadas por \, añade la siguienté así a continuación con \n invisible.
Y borra el \ y el \n por nada en ambas líneas quedando de seguido.


Espacio de almacenamiento:

h/H: h, copia el contenido del espacio de patrones en el espacio de almacenamento.
H hace lo mismo pero añadiendo un salto de líinea antes.


g/G: g copia el contenido del espacio de almacenamiento en el espacio de patrones, sobreescribiendo
lo que tuviese en ese momento. El comando G hace lo mismo añadiendo antes un salto de línea.


x(eXchange): Intercambia lo que hay en el espacio de patrones con lo que hay en el espacio
de almacenamiento.


EJEMPLO: sed '12,20{H;d} ; ${G}' texto.txt
De la linea 12 a la 20 se copia el contenido del espacio de patrones en el de almacenamiento con un salto de línea previo
y se elimina del espacio de patrones.

Luego al contenido de la última fila se le añaden las líneas 12 a 20 con un salto de linea previo(\n).
Con esto conseguimos que las líneas 12 a 20 salgan después de la última fila.
Técnicamente es una súper fila dividida en varias, pero visualmente pasa eso.

-Práctica 6 : Teoría

AWK es igual que el sed. Editor de flujo de texto, opera sobre un flujo de datos de entrada y lo
manda cara la salida. Pero awk está pensado para trabajar con flujos estructurados, donde
cada línea(registro) está organizada en campos.
El mejor ejemplo sería un archivo de texto organizado en filas y columnas, es decir, en formato
tabla.

En realidad AWK es un lenguaje de programación completo, con funciones, variables, sentencias de
control...

Los programas awk constan de unas reglas con el siguiente formato:
patrón {acción};patrón {acción};...

Un ejemplo básico sería:
awk '/clave/ {print $0}' file.txt
$0 hace que salga toda la línea.

En este ejemplo el patrón va entre comas simples. En este cso es /clave/ y buscamos lineas que
tengan ese patrón entero y las imprimimos enteras.

Por defecto awk asume que los campos están separados por espacios-tabuladores. Por lo que
los campos coinciden con las palabras.

Así como $0 es toda la línea, $1 es el primer campo, $2 el segundo y etc...

Por lo tanto en el ejemplo anterior, para sólo enseñar la primera palabra de cada línea deberiamos
hacer: awk '/clave/ {print $1}' file.txt.

Ahora veamos un ejemplo con dos reglas:
awk '/clave/ {print $0};/---/ {print $2}' file.txt

Por último indicar que existen dos patrones muy particulares representados por las palabras 
BEGIN y END. Las acciones indicadas por el patrón BEGIN se procesaran antes de la primera línea
del archivo. Permite por ejemplo imprimir una línea de cabecera en la salida. Las acciones
indicadas por el patrón END se realizarán después de la última línea del archivo. Veamos
un ejemplo:
awk ’BEGIN{print "Liñas con clave"};/clave/ {print $0}’ file.txt

Operadores:

1.Operadores aritméticos:+,-,*,\,%,++,--

2.Operadores de asignación:=. Como awk permite crear variables internas con este operador podemos asignarle valores
a dichas variables. Puede concatenarse con valores aritméticos. Ejemplo: x+=2.

3.Operadores condicionales:==,!=,>,>=,<,<=

4.Comparar expresiones regulares:~, !~. Permiten comparar si la cadena tiene o no la expresión. La normal saca
las líneas que la tienen, la otra las que no.
awk ’$0~/x$/ {print $0}’ file.txt : imprime las líneas que acaben en x.

5.Operadores lógicos:&&, ||, !

Ejemplos:
awk ’$1>100 {print $0}’ file.txt: Imprime las líneas donde el primer campo sea > 100.

awk ’$1 ~ /[0-9]{3}/ || $1 == 10 {print $0}’ file.txt:
Imprime todas las lineas donde el primer campo tenga justo 3 números, o en las que el primer campo sea igual a 10.


Variables:

Como ya dijimos awk permite usar variales internas como cualquier otro lenguaje de programación, sin iniciarlas ni
definir el tipo previamente.
Ejemplo:
awk ’BEGIN {suma=0};/^1/ {suma+=$1};END {print "Suma=",suma}’ file.txt
Por cada línea que empiece por 1, se le suma el contenido del campo 1 a la variable suma.
Al acabar la ejecución se imprime el resultado.

Además de las definidas por el usuario, AWK maneja una serie de variables internas muy útiles.

Las más importantes:

1.FS: Abreviatura de field separator. Por defecto awk considera que los campos están separados por espacios o
tabulados, pero estpuede cambiarse asignando otro valor a esta variable.
Por eemplo si queremos obtener un listado de los usuarios normales definidos en el sistema, procesando
o arquivo /etc/passwd, cuyos campos estén separados por el carácter ":", podríamos hacerlo así:
awk ’BEGIN{FS=":"};$3>999 {print $1,$3}’ /etc/passwd

2.NF: Abreviatura de number of fields: Contiene el número de campos de línea que se están procesando. Variable
muy útil cuando necesitamos hacer bucles, recorriendo los distintos campos, pero de momento vamos a poner un ejemplo
muy sencillo: 
awk ’BEGIN{FS=":"};$3>999 {print $1,$3,$NF}’ /etc/passwd
Lo mismo que antes, pero como NF es el número de campos de la línea estaríamos sacando el valor del campo 7.

3.NR: Abreviatura de number of records: Contiene el número de registro de la línea que se está procesando.
awk ’NR<10 {print $0}’ file.txt: Se sacarán por pantalla las líneas de la 1 a la 9.

4.OFS: Abreviatura de output field separator. Cunado ordenamos imprimir una serie de valores separados por ","
AWK interpreta "," como el separador de campo. Indica que es lo que va a separar a los campos que salgan por pantalla.
Ejemplo: 
awk ’BEGIN{FS=":";OFS=","};$3>999 {print $1,$3,$NF}’ /etc/passwd
Los campos saldrán separados por comas.



Sintaxis completa:

awk [opciones] [rograma] archivo(s)

Sino se pasan archivos sobre los que operar la entrada es estándar. Pero es muy común utilizar
el comando awk con tuberías para procesar o filtrar la salida de otros comandos:
ls -l | awk ’$6 == "nov" {sum += $5} END{print sum}’
Filtra la salida de ls -l y por cada línea que su campo 6 sea exactamente nov, se le suma a
una variable suma el campo 5. Al finalizar imprime la variable suma.
*NO ES NECESARIO INICIALIZAR LAS VARIABLES EN AWK, ASUME QUE TODAS SE INICIALIZAN EN 0.

Como ya indicamos anteriormente el programa tendrá varias instruciones con el formato
patrón {acción}, separadas por ;. Cuando trabajamos con programas complejos eiste la posibilidad
de guardar el programa en un archivo separado que acaba en la extensión .awk.
En este caso las distintas instrucciones pueden escribirse en distintas líneas.
Después usaremos una de las opciones que indicaremos a continuación para indicar que el programa
se lea de ese archivo.

Opciones importantes:
1. -f programa, pedimos que el programa awk se lea del archivo programa.
2. -F separador, es otra manera de modificar el separador de campos por defecto.
Ejemplo: awk -F ':'.
3.-vvar=valor, Con esta opción podeos pasar datos al programa awk.

Ejemplo simple:
Vamos a modificar elejemplo anterior(awk ’$6 == "nov" {sum += $5} END{print sum}’)
para escribirlo en un archivo y permitir que el mes específico sea pasado como
una variable a awk.

Escribimos lo siguiente en un archivo vi llamado por ejemplo espacio_mes.awk.

BEGIN{print "Espacio ocupado por los archivos del mes: " mes}
$6 == mes {sum+=$5}
END{print "Total= "sum}

Luego podemos invocar este programa por ejemplo de la siguiente manera desde la línea de comandos:

ls -l | awk -vmes="feb" -f espacio_mes.awk


Más cosas:
Para acabar vamos a presentar una serie de ejemplos más complejos, que nos servirán para introducir
más las características del comando AWK. 
Entre ellas la posibilidad para especificar patrones de busca, uso de bucles y finalmente
algo más avanzado pero potente, los arrays asociativos.

Ejemplos:

1.Filtrar las lineas en blanco de un archivo.

2.Enseñar el listado de grupos que tienen más de un usuario.

> 1 { print $1, "->", num_membros, "membros" }' /etc/group 

-F dice que separacion viene el archivo de entrada. Lla variable num_miembros cuenta el número con la función split que almacena
el número de usuarios por grupo(campo4) en ese array, y lo hace contando cuantas cosas hay entre comas,
luego comprueba si el número es mayor a 1, y en caso de que si, imprime el nombre del grupo
junto al número de miembros.

3.Enseñar la lista de comandos más usados, ordenados por la frecuencia de uso.

4.Extraer la función main de un programa en C.
awk ’/main/,/^}/’ prog.c
